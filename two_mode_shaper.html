Multimode input shaper (prototype / concept)

When an axis has multiple separated resonances, a standard shaper forces a choice: tune to one peak and leave the rest ringing, or use a broadband smoother wide enough to blanket the whole span. A multimode shaper — the convolution of a single-mode shaper tuned to each peak — places a notch at every configured frequency; the residual is the product of each component's residual, so it drives all of them toward zero at once. This is Kalico's real shaper_type = multimode (config keys shaper_base_<axis> / shaper_freq_<axis> / damping_ratio_<axis>, each a comma-separated list). Base shapers below are restricted to the six impulse shapers shaper_base_<axis> actually accepts.

Measured on a real printer: before vs after (actual accelerometer data)

  
The same physical machine measured twice: a SHAPER_CALIBRATE capture with no shaper, against a raw accelerometer capture from the identical resonance sweep with the tuned multimode shaper active. The raw capture is processed with Kalico's exact pipeline (Welch PSD: Kaiser(6) window, 50% overlap, per-window detrend — shaper_calibrate.calc_freq_response) and put in the same normalized units as the calibration file (normalize_to_frequencies). Sanity check: at 1–2 Hz, far below any resonance, the two captures agree within a few percent — so the excitation was equivalent and the comparison is apples-to-apples.

Multimode shaper = convolution of a base shaper tuned to each configured peak (amplitudes multiply, times add); residual computed with the classic amplitude formula (see the input shaper page). The broadband reference is smooth_zvd_ei tuned to peak 1, the widest existing smoother. Manual config (shaper_freq_<axis> = f1, f2, f3) supports any N ≥ 2 up to the firmware's 32-impulse pulse buffer; SHAPER_CALIBRATE's own automatic search still only tests the top two detected peaks. The fitted extruder kernel reproduces Kalico's real fitting procedure (Legendre-basis, KKT-constrained least squares with a soft non-negativity penalty, evaluated near every peak with that peak's own damping) — the same algorithm as extruder_smoother.get_multi_mode_extruder_smoother.
